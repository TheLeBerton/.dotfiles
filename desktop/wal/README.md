# WAL — Unified Color Theming

## Why

Before, changing colorscheme meant manually editing each app's config separately. Too many files, too much repetition, and often inconsistencies between apps.

This setup unifies everything through pywal: one single source of truth for colors, and all apps follow automatically. Only 2-3 files to touch to add a new theme.

---

## How it works

**The flow:**
1. **Choose** — launch `theme.sh` via rofi, pick a dark/light theme
2. **Generate** — pywal reads the JSON colorscheme and generates all cache files in `~/.cache/wal/`
3. **Apply** — `apply-colors.sh` reloads each app with the new colors

**How each app consumes the colors:**

| App | Method | Particularity |
|-----|--------|---------------|
| Hyprland | sources `pencils-hyprland.conf` | also applied live via `hyprctl` without restart |
| Waybar | CSS `@import` | needs full restart to reload |
| Kitty | `include` in kitty.conf | pywal has built-in support, no custom template needed |
| Rofi | `@import` in `.rasi` | reads on each launch |
| Neovim | watches `nvim-theme` file | updates live without restarting nvim |
| Firefox | `pywalfox update` | external tool, not native wal |
| Swaync | `swaync-client --reload-css` | CSS reload, no restart needed |

---

## Cache files

All generated files live in `~/.cache/wal/`. Each app reads its own file:

| File | Consumed by |
|------|-------------|
| `colors.sh` | `apply-colors.sh` (bash vars) |
| `colors-neovim.lua` | Neovim via `wal.lua` |
| `colors-waybar.css` | Waybar via `@import` |
| `colors-swaync.css` | Swaync via reload |
| `colors.rasi` | Rofi via `@import` |
| `pencils-hyprland.conf` | Hyprland via `source =` |
| `colors-kitty.conf` | Kitty via `include` |
| `nvim-theme` | Neovim file watcher |

> These files are **never edited manually** — always regenerated by pywal.

---

## How to switch themes

Triggered via `META+R` which launches `menu.sh`, which opens rofi.
From there, select **Theming** to open `theme.sh`.

The script will ask:
1. Dark or Light mode
2. Which theme

Then automatically: updates the wallpaper, regenerates all cache files, and reloads all apps.

---

## How to add a new theme

1. Add the wallpaper image to your wallpapers directory
2. Create the colorscheme JSON in `desktop/wal/colorschemes/dark/` or `light/`
3. Add an entry to `THEME_DATA` in `theme.sh` with the 5 fields:
   - **wal-theme** — name of the JSON file (without extension)
   - **wallpaper** — image filename
   - **rounding** — corner rounding in px (`0` = sharp, `10` = rounded)
   - **waybar-style** — `squared.css` or `style.css`
   - **nvim-theme** — colorscheme name passed to Neovim
4. Add the theme name to `DARK_THEMES` or `LIGHT_THEMES` array in `theme.sh`
